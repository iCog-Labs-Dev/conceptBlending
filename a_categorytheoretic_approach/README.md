# Category-Theoretic Concept Blending

This module implements a **Neuro-Symbolic** approach to conceptual blending. It leverages the semantic reasoning capabilities of Large Language Models (LLMs) alongside the mathematical rigor of **Category Theory (Colimits / Pushouts)** to generate deterministic, logically sound, and highly compressible conceptual blends.

---

## Architecture: The 4-Phase Pipeline

The system operates as a multi-agent pipeline where the LLM acts as the *semantic bridge* and the Python backend acts as the *symbolic engine*.

### Phase 1: Specification Generation
The LLM translates raw concepts (e.g., `"House"` and `"Boat"`) into formal **Algebraic Specifications** (S-Expressions) containing:
- `sorts`
- `operations`
- `predicates`
- `axioms`

### Phase 2: Generalization Search (Generic Space)
The system identifies the semantic overlap, known as the **Generic Space** ($G$), which acts as the shared interface between Concept A and Concept B.

### Phase 3: Morphism Mapping
Structure-preserving maps (morphisms) are drawn from:

$G \to A$  
$G \to B$

This alignment informs the symbolic engine how the concepts correspond structurally.

### Phase 4: Colimit Computation (The Blend)
A purely deterministic mathematical operation (the **Pushout**) unifies the graphs, resolves conflicts, and outputs the final blended concept.

## Evaluation & Metrics Framework

This module includes a research-grade evaluation framework that measures both:

- **Quality of the Product** (the blend itself)
- **Efficiency of the Pipeline** (the system performance)

---

### 1. Quality Metrics (The Blend)

Computed automatically during Phase 4:

- **Richness (InfoValue)**  
  Total semantic weight of the resulting blend.

- **Synergy (Compression)**  
  Degree of structural integration between shared features.

- **Fidelity (SFS — Structural Fidelity Score)**  
  Measures logical preservation (1.0 = no input axioms violated).

- **Efficiency (CCR — Conceptual Compression Ratio)**  
  Blend compactness relative to the sum of inputs.

- **Balance**  
  Symmetry of contribution between Concept A and Concept B.

---

### 2. System Performance (The Pipeline)

Tracked via `libs/performance_monitor.py`:

- **End-to-End Latency Breakdown**  
  Separates LLM (Neuro) time from Symbolic Engine time.

  > Note: The symbolic colimit computation typically executes in < 0.005 seconds.

- **Cognitive ROI**


  Synergy *  Fidelity / Latency


- **Syntactic Validity Rate (SVR)**  
  Percentage of valid, parseable S-Expressions generated by the LLM.
